## Design Context

### Users
Solo developers managing their own AI agent skills across multiple agents. They open Anvil to get things done efficiently — install, edit, organize skills — then close it. The app should feel like a well-made tool that stays out of the way.

### Brand Personality
**Effective, simple, clear.** Anvil is a precision instrument, not a showcase. Every element earns its place. The interface communicates through structure and clarity rather than decoration or delight.

### Aesthetic Direction
- **Visual tone**: The Forge. Warm amber palette (hue 52°) — every neutral carries a subtle ember warmth, like iron cooling in a workshop. Color is used sparingly: amber for brand/primary actions, semantic status colors for audits and feedback, destructive red for danger.
- **Typography**: Space Grotesk Variable for headings (geometric, distinctive), Geist Variable for body text (clean, technical). Headings use tight negative tracking (-0.03em to -0.01em). Uppercase labels use 0.08em letter-spacing via `.label-caps`.
- **Radius**: 0.375rem — sharp and precise, tool-like. Derived scale via CSS calc (0.6×–2.6×).
- **Shadows**: Layered depth shadows in light mode (warm-tinted, multiple offsets at low opacity). Amber border-glow in dark mode. Five levels: sm, md, lg, elevated, dreamy.
- **Reference**: Linear — information-dense without feeling heavy, fast keyboard-driven interactions, restrained use of color, typography does the heavy lifting.
- **Anti-reference**: AI slop — no gratuitous gradients, no glowing orbs, no "magical" purple-to-blue washes, no floating particles, no "powered by AI" aesthetic. The app manages AI tools but should not look like an AI marketing page.
- **Theme**: Light and dark mode, system-preference aware. OKLCH color space throughout for perceptual uniformity. All neutrals tinted with hue 52–68° (never pure gray).

### Micro-interactions
- **Card hover**: translateY(-2px) + shadow intensification (sm → elevated), 200ms ease
- **Button press**: scale(0.97) on active, 80ms ease
- **Input focus**: amber ring glow (4px spread at 12%/18% opacity light/dark)
- **Selection**: amber-tinted highlight (25%/30% opacity light/dark)
- **Reduced motion**: all motion disabled via `prefers-reduced-motion: reduce`

### Design Principles
1. **Density over sprawl** — Show more information in less space. Prefer compact, scannable layouts. No hero sections, no excessive whitespace padding.
2. **Structure communicates** — Use typography hierarchy, spacing, and alignment to convey meaning. Avoid relying on color, icons, or labels when layout alone can make relationships clear.
3. **Quiet confidence** — Micro-interactions are subtle and functional (hover feedback, press confirmation), never decorative. No skeleton loaders for instant operations, no unnecessary loading states. The app should feel fast and certain.
4. **Keyboard-first** — Every primary action reachable by keyboard. Command palette as the power-user hub. Mouse is supported but not privileged.
5. **Earn every pixel** — If an element doesn't help the user complete a task, remove it. No decorative dividers, no placeholder illustrations, no "empty state" artwork.
