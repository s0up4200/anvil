## Design Context

### Users
Solo developers managing their own AI agent skills across multiple agents. They open Anvil to get things done efficiently — install, edit, organize skills — then close it. The app should feel like a well-made tool that stays out of the way.

### Brand Personality
**Effective, simple, clear.** Anvil is a precision instrument, not a showcase. Every element earns its place. The interface communicates through structure and clarity rather than decoration or delight.

### Aesthetic Direction
- **Visual tone**: Calm control. Minimal, achromatic palette (grayscale + purple brand accent) is intentional — color is used sparingly and purposefully, primarily for agent brand indicators and destructive actions.
- **Reference**: Linear — information-dense without feeling heavy, fast keyboard-driven interactions, restrained use of color, typography does the heavy lifting.
- **Anti-reference**: AI slop — no gratuitous gradients, no glowing orbs, no "magical" purple-to-blue washes, no floating particles, no over-animated transitions, no "powered by AI" aesthetic. The app manages AI tools but should not look like an AI marketing page.
- **Theme**: Light and dark mode, system-preference aware. OKLCH color space for perceptual uniformity.

### Design Principles
1. **Density over sprawl** — Show more information in less space. Prefer compact, scannable layouts. No hero sections, no excessive whitespace padding.
2. **Structure communicates** — Use typography hierarchy, spacing, and alignment to convey meaning. Avoid relying on color, icons, or labels when layout alone can make relationships clear.
3. **Quiet confidence** — No flashy animations, no skeleton loaders for instant operations, no unnecessary loading states. The app should feel fast and certain.
4. **Keyboard-first** — Every primary action reachable by keyboard. Command palette as the power-user hub. Mouse is supported but not privileged.
5. **Earn every pixel** — If an element doesn't help the user complete a task, remove it. No decorative dividers, no placeholder illustrations, no "empty state" artwork.
